Detailed Description

CTS_NVM_drv Source.

This driver provides a default interface for storing and retreiving non-volatile configuration settings. It assumes the configuration structure is the only thing that is stored, thus no addressing is required apart from the base address definition.

Clear current NVM storage, and store new page.

Clear NVM storage, and writes a default header.

Writes a CTS NVM page to the NVM without retreiving and checking existing data in the NBM. Makes sure the page is stored valid so that a next CTS_NVM_Store operation succeeds, given that the NVM doesnt cause any errors.

Parameters
dataBuffstructure to store in default frame
dataLengthsize of the structure to store on a CTS_NVM_Store command, used for read validation.
Return values
CTS_NVM_WRITE_FAILEDSomething went wrong writing to NVM
CTS_NVM_PAGE_INVALIDDefault page readback failed.
CTS_NVM_OKConfiguration was succesfully stored
